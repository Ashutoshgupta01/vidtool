General

  1. Be nice to everybody.
  2. Respect the hard work of other developers.
  3. Please share quality projects.
  4. Respect the admins and moderators. Don’t argue with them, they are just doing their job. They are volunteers trying to keep Sketchub safe.
  5. If someone is violating the user policy, please show them this page.
              
                If you have any complaints or suggestions regarding Sketchub, feel free to drop a message at [email protected]

Projects

  1. Overall
    1. Project details should be original and detailed.
    2. Please don’t use inappropriate contents, like:
      1. Insults, hatred, racism, etc.
      2. NSFW content (Like nudity or graphical violence).
      3. Unrelated subjects
    3. Projects should not contain malware, crappy ads, frauds, phishing, etc.
    4. Projects related to particular mod (Like game hacks) are not allowed.
    5. Please avoid using copyrighted content in your project and your project’s listing, that includes copyrighted screenshots and logo.
    6. Please don’t promise a reward (like making the app open source) once your project reaches a certain number of likes or comments.
  2. Graphics (Icons/Screenshots)
    1. Don’t use pictures that aren’t related to your project.
    2. Don’t use someone else’s pictures.
    3. Don’t use really bad icons, like a bright green icon to distract people browsing apps.
  3. Project files
    1. Your project shouldn’t be a stolen project.
    2. You are only eligible to upload someone’s project if:
      1. You made changes (like improved code, fixed bugs, added features)
      2. The original project is uploaded on Sketchub
      3. You attributed the original creator.
      4. You are uploading the project on behalf of someone else.
    3. You are NOT allowed to upload someone’s project if you only change project name and theme colors with no special efforts.
    4. Please note that by uploading a project, everyone can reupload it according to line number 1.c.ii
    5. Your project shouldn’t be useless, it should have some functionality.
    6. If it is Sketchware Project, it shouldn't have more than 10 compile errors.
    7. Please do not upload non-compilable resource alone or force users to install them, that includes APK, AAB and EXE. (Dont confuse it with app store or project stores, they are still allowed).
  4. If you want to know if your project is eligible for verification or Editor’s Choice, please check our Forum pages.

Types of projects that are not allowed

  1. Projects that generates fake bank cards
  2. Youtube viewer generators

Some exceptions for project categories/types

  1. Less functional project can be uploaded as "Example & Tutorial" category (But it should not be really common project, like simply setting text to a textview, etc.)
  2. Project with just designs/widgets (and no backend) can be published as UI/UX

User account/profile

  1. Don’t impersonate someone else.
  2. Don’t add anything inappropriate to your profile.
  3. Don’t create multiple accounts.
  4. If you are banned, appeal us at [email protected]
  5. If someone is violating the user policy, please show them this page.

Comments

  1. Please don’t use comments to say something that’s off-topic.
  2. You are only allowed to advertise in your own project’s comment section.
  3. Please don’t interfere with staff members when they are trying to do their job. By doing so, you are making their job harder. Stay out of their way until the situation has been resolved.
  4. Please refrain from asking the publisher to upload the project of a closed source program.
  5. Don't ping/mention anyone unnecessarily.

Reports

  1. Reports are taken seriously, so please make sure that you’re reporting something for the right reason before you submit it.
  2. Please don’t report projects for being stolen unless you are the owner of the project and the project you’re reporting doesn’t follow these rules: 1.c.ii of Projects.
  3. After reporting anything, you might receive an email. Please reply with a proper message if it is related to you and you want your report to be taken seriously.
  4. All projects are free to use, but they are not always ready to use, so please don’t report projects that aren’t compiling.
